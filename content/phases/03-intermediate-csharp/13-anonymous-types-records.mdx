## Anonymous Types

Anonymous types are compiler-generated, immutable, read-only reference types — primarily used in LINQ `select` projections:

```csharp
// Anonymous type — compiler generates a class with read-only properties
var user = new { Name = "Alice", Age = 30, IsAdmin = true };

Console.WriteLine(user.Name);  // Alice
Console.WriteLine(user.Age);   // 30

// user.Name = "Bob"; // COMPILE ERROR — anonymous type properties are read-only

// Common use: LINQ projection to reduce data
var products = new[]
{
    new { Id = 1, Name = "Widget", Price = 9.99m, Cost = 4.00m, Stock = 100 }
};

var summary = products
    .Where(p => p.Stock > 0)
    .Select(p => new { p.Id, p.Name, Margin = p.Price - p.Cost }) // anonymous projection
    .ToList();

// summary[0].Id = 1, summary[0].Name = "Widget", summary[0].Margin = 5.99
```

### Limitations of Anonymous Types

- Cannot be returned from methods (except as `object` or `dynamic`)
- Cannot be used as method parameters (without `object`/`dynamic`)
- Compiler equality: two anonymous type instances with same properties and values ARE equal

```csharp
var a = new { X = 1, Y = 2 };
var b = new { X = 1, Y = 2 };
Console.WriteLine(a.Equals(b)); // True — value equality (same type, same values)
Console.WriteLine(a == b);      // False — == uses reference equality for classes!
```

## Records (C# 9+)

Records are a concise syntax for immutable data types with **value equality** built in:

```csharp
// Positional record — compact syntax, primary constructor
public record Point(int X, int Y);

// Equivalent expanded form (what the compiler generates):
public record PointExpanded
{
    public int X { get; init; }
    public int Y { get; init; }

    public PointExpanded(int x, int y) { X = x; Y = y; }

    // Auto-generated: Equals, GetHashCode, ToString, ==, !=, Deconstruct
}

// Usage
var p1 = new Point(1, 2);
var p2 = new Point(1, 2);
var p3 = new Point(3, 4);

Console.WriteLine(p1 == p2);    // True  — VALUE equality (compares X and Y)
Console.WriteLine(p1 == p3);    // False
Console.WriteLine(p1.ToString()); // Point { X = 1, Y = 2 }

// Deconstruction — built in for positional records
var (x, y) = p1;
Console.WriteLine($"x={x}, y={y}"); // x=1, y=2
```

## With Expressions: Non-Destructive Mutation

Records are immutable — but you can create modified copies with `with`:

```csharp
public record Address(string Street, string City, string Country);
public record Person(string Name, int Age, Address HomeAddress);

var original = new Person("Alice", 30, new Address("123 Main St", "London", "UK"));

// 'with' creates a shallow copy with specified properties changed
var moved = original with { HomeAddress = original.HomeAddress with { City = "Manchester" } };

Console.WriteLine(original.HomeAddress.City); // London   — original unchanged
Console.WriteLine(moved.HomeAddress.City);    // Manchester

// original and moved are different objects:
Console.WriteLine(ReferenceEquals(original, moved)); // False
Console.WriteLine(original == moved);                 // False (City differs)
```

## Init-Only Properties (C# 9)

The `init` accessor allows a property to be set during object initialization but not afterwards:

```csharp
public class Order
{
    // init-only — set in object initializer, immutable after
    public int    Id      { get; init; }
    public string Product { get; init; } = "";
    public decimal Price  { get; init; }

    // Regular setter — mutable
    public string Status { get; set; } = "Pending";
}

var order = new Order
{
    Id      = 1,
    Product = "Widget",
    Price   = 9.99m
};

order.Status = "Shipped"; // OK — regular setter
// order.Id = 2;          // COMPILE ERROR — init-only property

// Works with object initializer patterns in tests:
var testOrder = order with { }; // records use init under the hood
```

<Callout type="senior">
**Senior Insight: Record Structs for Zero-Allocation Value Semantics**

C# 10 added `record struct` — a value type with auto-generated value equality and `with` support, but no heap allocation:

```csharp
// record struct — value type + value equality + with
public record struct Money(decimal Amount, string Currency);

var price = new Money(9.99m, "USD");
var discounted = price with { Amount = 7.99m };

Console.WriteLine(price);       // Money { Amount = 9.99, Currency = USD }
Console.WriteLine(discounted);  // Money { Amount = 7.99, Currency = USD }
Console.WriteLine(price == discounted); // False

// No heap allocation — Money lives on the stack like any struct
// Ideal for: coordinates, currency, temperatures, colors, IDs

// readonly record struct — immutable value type (C# 10)
public readonly record struct Point(int X, int Y);
// All properties are init-only; the struct cannot be mutated
```

Choosing between:
- `record` (class): domain entities, DTOs, anything passed by reference or needing inheritance
- `record struct`: small immutable value objects, high-performance scenarios (geometry, finance math)
- `readonly record struct`: same as record struct but enforces immutability at the compiler level
</Callout>
