## The Problem: You Need Indirect Access

Sometimes you need to intercept access to an object to:
- **Delay creation** (lazy initialization) of an expensive object
- **Control access** (authorization/security check before delegation)
- **Log calls** (audit trail without modifying the real object)
- **Cache results** (avoid repeated expensive calls)
- **Remote access** (RPC — talk to an object on another machine)

Proxy implements the same interface as the real object — clients don't know they're talking to a proxy.

## Virtual Proxy (Lazy Loading)

```csharp
public interface IDocumentLoader
{
    string GetContent();
    int    GetPageCount();
}

// Real object — expensive to create (loads from disk/DB)
public class PdfDocument : IDocumentLoader
{
    private readonly string _path;
    private readonly byte[] _data; // expensive — loaded immediately

    public PdfDocument(string path)
    {
        _path = path;
        Console.WriteLine($"Loading PDF from {path}..."); // expensive
        _data = File.ReadAllBytes(path);
    }

    public string GetContent()   => System.Text.Encoding.UTF8.GetString(_data);
    public int    GetPageCount() => _data.Length / 4096; // simplified
}

// Virtual proxy — defers creation until first use
public class LazyPdfProxy : IDocumentLoader
{
    private readonly string _path;
    private PdfDocument? _document; // null until needed

    public LazyPdfProxy(string path) => _path = path;

    private PdfDocument Document => _document ??= new PdfDocument(_path); // lazy init

    public string GetContent()   => Document.GetContent();
    public int    GetPageCount() => Document.GetPageCount();
}

// Usage — PDF is NOT loaded until you actually access content
IDocumentLoader doc = new LazyPdfProxy("report.pdf");
Console.WriteLine("Proxy created — no PDF loaded yet");
// Only NOW does the PDF load:
int pages = doc.GetPageCount();
```

## Protection Proxy (Access Control)

```csharp
public interface IAdminService
{
    void DeleteUser(int userId);
    void ResetPassword(int userId, string newPassword);
}

public class AdminService : IAdminService
{
    public void DeleteUser(int userId)
        => Console.WriteLine($"Deleted user {userId}");

    public void ResetPassword(int userId, string newPassword)
        => Console.WriteLine($"Reset password for {userId}");
}

// Protection proxy — checks permissions before delegating
public class AuthorizationProxy : IAdminService
{
    private readonly IAdminService       _inner;
    private readonly IHttpContextAccessor _http;

    public AuthorizationProxy(IAdminService inner, IHttpContextAccessor http)
    {
        _inner = inner;
        _http  = http;
    }

    private void RequireRole(string role)
    {
        if (_http.HttpContext?.User.IsInRole(role) != true)
            throw new UnauthorizedAccessException($"Role '{role}' required.");
    }

    public void DeleteUser(int userId)
    {
        RequireRole("SuperAdmin");  // check BEFORE delegating
        _inner.DeleteUser(userId);
    }

    public void ResetPassword(int userId, string newPassword)
    {
        RequireRole("Admin");
        _inner.ResetPassword(userId, newPassword);
    }
}
```

## Real .NET Example: EF Core Lazy Loading Proxies

```csharp
// Enable lazy loading proxies in EF Core
builder.Services.AddDbContext<AppDbContext>(options =>
    options.UseSqlServer(connectionString)
           .UseLazyLoadingProxies()); // EF generates proxy classes at runtime

public class Order
{
    public int    Id         { get; set; }
    public string Status     { get; set; } = "";
    public virtual Customer Customer { get; set; } = null!; // virtual = proxyable
    public virtual ICollection<OrderItem> Items { get; set; } = new List<OrderItem>();
}

// EF returns a proxy subclass of Order, not Order itself
var order = await dbContext.Orders.FindAsync(1);
// order is actually OrderProxy : Order — EF generated it

// Accessing the navigation property triggers a lazy DB query via the proxy
Customer customer = order.Customer; // proxy intercepts — runs SELECT * FROM Customers WHERE Id = ...

// The proxy pattern is transparent to your code
Console.WriteLine(customer.Name); // works normally
```

## Dynamic Proxy with `DispatchProxy`

`DispatchProxy` lets you create proxies at runtime without subclassing:

```csharp
// A logging proxy for any interface — generated at runtime
public class LoggingProxy<T> : DispatchProxy
{
    private T? _target;
    private ILogger? _logger;

    // Initialize with the real target and logger
    public static T Create(T target, ILogger logger)
    {
        var proxy = Create<T, LoggingProxy<T>>(); // creates the proxy
        var p = (LoggingProxy<T>)(object)proxy;
        p._target = target;
        p._logger = logger;
        return proxy;
    }

    // All method calls go through Invoke
    protected override object? Invoke(MethodInfo? targetMethod, object?[]? args)
    {
        _logger!.LogInformation("Calling {Method}", targetMethod?.Name);
        try
        {
            var result = targetMethod!.Invoke(_target, args);
            _logger.LogInformation("{Method} completed", targetMethod.Name);
            return result;
        }
        catch (TargetInvocationException ex)
        {
            _logger.LogError(ex.InnerException, "{Method} failed", targetMethod?.Name);
            throw ex.InnerException!;
        }
    }
}

// Usage — create a logging proxy for any interface
IOrderRepository realRepo = new SqlOrderRepository(dbContext);
IOrderRepository loggingProxy = LoggingProxy<IOrderRepository>.Create(realRepo, logger);

await loggingProxy.GetByIdAsync(42); // logged automatically!
```

<Callout type="senior">
**Castle.DynamicProxy — AOP in .NET**

`DispatchProxy` is limited to interface proxies. Castle.DynamicProxy (used by Moq, NSubstitute, and many DI containers) can proxy both interfaces and virtual class methods, and is far more capable:

```csharp
// Castle.DynamicProxy — used by most mocking libraries under the hood
var generator = new ProxyGenerator();

// Create an interface proxy with an interceptor
IOrderRepository proxy = generator.CreateInterfaceProxyWithTarget(
    target: new SqlOrderRepository(dbContext),
    interceptors: new IInterceptor[]
    {
        new LoggingInterceptor(logger),
        new CachingInterceptor(cache),
    });

public class LoggingInterceptor : IInterceptor
{
    private readonly ILogger _logger;
    public LoggingInterceptor(ILogger logger) => _logger = logger;

    public void Intercept(IInvocation invocation)
    {
        _logger.LogInformation("Calling {Method}", invocation.Method.Name);
        invocation.Proceed(); // call the next interceptor or the target
        _logger.LogInformation("{Method} returned: {Value}", invocation.Method.Name, invocation.ReturnValue);
    }
}
```

Castle.DynamicProxy is used internally by:
- **Moq** and **NSubstitute** (test mocking)
- **Autofac** and **Ninject** (interception in DI)
- **EF Core** (lazy loading proxies)

Understanding this helps explain "how does Moq work?" — it creates a Castle proxy and records/replays calls.
</Callout>
