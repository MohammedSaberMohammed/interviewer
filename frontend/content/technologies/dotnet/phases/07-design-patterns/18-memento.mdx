## The Problem: Saving and Restoring State

You're building a text editor with undo. You need to save snapshots of the document's state — but you don't want the undo history manager to access or manipulate the document's internal fields directly (breaking encapsulation).

Memento solves this: the document (Originator) creates a Memento containing a snapshot of its state. The CareTaker (history manager) stores the Memento without knowing what's inside it.

---

## Structure

```csharp
// Memento — stores a snapshot of Originator's state
// Its fields are opaque to the CareTaker
public sealed class DocumentMemento
{
    internal string Content  { get; }
    internal int    CursorPosition { get; }
    internal string FontName { get; }

    // Internal constructor — only DocumentEditor can create mementos
    internal DocumentMemento(string content, int cursor, string font)
    {
        Content        = content;
        CursorPosition = cursor;
        FontName       = font;
    }
}

// Originator — the object whose state we want to capture
public class DocumentEditor
{
    private string _content  = "";
    private int    _cursor   = 0;
    private string _fontName = "Arial";

    public void InsertText(string text)
    {
        _content = _content.Insert(_cursor, text);
        _cursor += text.Length;
    }

    public void SetFont(string font) => _fontName = font;

    // Save current state — creates a Memento
    public DocumentMemento Save()
        => new DocumentMemento(_content, _cursor, _fontName);

    // Restore from a Memento
    public void Restore(DocumentMemento memento)
    {
        _content  = memento.Content;
        _cursor   = memento.CursorPosition;
        _fontName = memento.FontName;
    }

    public void Print() =>
        Console.WriteLine($"Content: '{_content}' | Cursor: {_cursor} | Font: {_fontName}");
}

// CareTaker — stores mementos, knows nothing about their internals
public class UndoHistory
{
    private readonly Stack<DocumentMemento> _history = new();
    private readonly Stack<DocumentMemento> _redoStack = new();

    public void Push(DocumentMemento memento)
    {
        _history.Push(memento);
        _redoStack.Clear();
    }

    public DocumentMemento? Undo() => _history.TryPop(out var m) ? m : null;

    public void PushRedo(DocumentMemento m) => _redoStack.Push(m);
    public DocumentMemento? Redo() => _redoStack.TryPop(out var m) ? m : null;
}

// Usage
var editor  = new DocumentEditor();
var history = new UndoHistory();

editor.InsertText("Hello");
history.Push(editor.Save()); // snapshot after "Hello"

editor.InsertText(" World");
history.Push(editor.Save()); // snapshot after "Hello World"

editor.SetFont("Times New Roman");
editor.Print(); // Content: 'Hello World' | Font: Times New Roman

// Undo font change
history.Push(editor.Save()); // save current before undo
var memento = history.Undo();
if (memento != null) editor.Restore(memento);
editor.Print(); // Content: 'Hello World' | Font: Arial
```

## Simplified with `record` — Immutable Snapshots

C# records make Memento trivial for value-like state:

```csharp
// The record IS the memento — immutable snapshot
public record EditorState(string Content, int CursorPosition, string FontName);

public class DocumentEditor
{
    public string Content        { get; private set; } = "";
    public int    CursorPosition { get; private set; } = 0;
    public string FontName       { get; private set; } = "Arial";

    // Create a snapshot (memento)
    public EditorState Snapshot()
        => new EditorState(Content, CursorPosition, FontName);

    // Restore from snapshot
    public void Restore(EditorState state)
    {
        Content        = state.Content;
        CursorPosition = state.CursorPosition;
        FontName       = state.FontName;
    }

    public void InsertText(string text)
    {
        Content        = Content.Insert(CursorPosition, text);
        CursorPosition += text.Length;
    }
}

// Undo manager with record snapshots
public class UndoManager
{
    private readonly Stack<EditorState> _undoStack = new();

    public void ExecuteWithUndo(DocumentEditor editor, Action<DocumentEditor> action)
    {
        _undoStack.Push(editor.Snapshot()); // save before action
        action(editor);
    }

    public void Undo(DocumentEditor editor)
    {
        if (_undoStack.TryPop(out var state))
            editor.Restore(state);
    }
}
```

## Real .NET Example: EF Core Change Tracking

EF Core's change tracker is an internal Memento system. When you load an entity, EF Core saves a snapshot of its original values:

```csharp
var order = await dbContext.Orders.FindAsync(orderId);
// EF Core internally saved: OriginalValues { Status: "Pending", Total: 100.00 }

order.Status = "Shipped";
order.Total  = 95.00m; // discount applied

// EF Core compares current values to original snapshot
var entry = dbContext.Entry(order);
Console.WriteLine(entry.OriginalValues["Status"]); // "Pending"
Console.WriteLine(entry.CurrentValues["Status"]);  // "Shipped"

// SaveChanges uses the diff to generate: UPDATE Orders SET Status='Shipped', Total=95 WHERE Id=...
await dbContext.SaveChangesAsync();

// You can even roll back to the snapshot
entry.CurrentValues.SetValues(entry.OriginalValues); // restore to "Pending", 100.00
```

<Callout type="senior">
**Serialization as Memento for Distributed Systems**

In distributed systems, snapshots are serialized to durable storage. This pattern underlies:
- **Saga checkpoints** — save saga state so it can resume after failure
- **Actor model persistence** — Akka.NET / Microsoft Orleans serialize actor state
- **Game save files** — serialize entire game state to disk

```csharp
// Serialization-based memento — persists across process restarts
public class OrderSaga
{
    public int    OrderId    { get; set; }
    public string State      { get; set; } = "Started";
    public bool   Paid       { get; set; }
    public bool   Shipped    { get; set; }

    // Serialize current state to a checkpoint
    public string CreateCheckpoint()
        => JsonSerializer.Serialize(this);

    // Restore from checkpoint
    public static OrderSaga RestoreFromCheckpoint(string json)
        => JsonSerializer.Deserialize<OrderSaga>(json)!;
}

// Store checkpoint after each step
await checkpointStore.SaveAsync(saga.OrderId, saga.CreateCheckpoint());

// On restart — resume from last checkpoint
string? checkpoint = await checkpointStore.GetAsync(orderId);
if (checkpoint != null)
    saga = OrderSaga.RestoreFromCheckpoint(checkpoint);
```

The key distinction from basic Memento: serialization-based snapshots survive process crashes, enabling exactly-once processing and durable workflow orchestration.
</Callout>
