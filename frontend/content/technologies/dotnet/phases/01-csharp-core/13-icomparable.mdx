## The Distinction in One Sentence

- **`IComparable<T>`**: implemented by the type itself — defines its *natural, intrinsic ordering*
- **`IComparer<T>`**: a separate class — defines an *external, pluggable ordering strategy*

Both produce a sorting order, but where the logic lives — and who controls it — is fundamentally different.

---

## IComparable\<T\>: The Type's Own Order

Implement `IComparable<T>` on a type when it has an obvious, universally agreed-upon ordering. `List<T>.Sort()`, `Array.Sort()`, and LINQ's `OrderBy` (without a key selector) rely on this interface.

`CompareTo` returns:
- **Negative** → this instance comes before `other`
- **Zero** → this instance has the same order as `other`
- **Positive** → this instance comes after `other`

```csharp
public class Product : IComparable<Product>
{
    public string Name { get; init; } = "";
    public decimal Price { get; init; }
    public int Stock { get; init; }

    // Natural order: by price ascending
    public int CompareTo(Product? other)
    {
        if (other is null) return 1; // null sorts before this
        return Price.CompareTo(other.Price);
    }

    public override string ToString() => $"{Name} (${Price})";
}

var products = new List<Product>
{
    new() { Name = "Widget", Price = 9.99m, Stock = 100 },
    new() { Name = "Gadget", Price = 4.99m, Stock = 50 },
    new() { Name = "Doohickey", Price = 14.99m, Stock = 25 },
};

products.Sort(); // Uses IComparable<Product>.CompareTo
foreach (var p in products)
    Console.WriteLine(p);
// Gadget ($4.99)
// Widget ($9.99)
// Doohickey ($14.99)

// Also works with Array.Sort, SortedList, SortedSet
var sorted = new SortedSet<Product>(products);
```

## IComparer\<T\>: External Sorting Strategies

`IComparer<T>` lets you define alternative orderings without modifying the type. This is the Strategy pattern applied to comparison:

```csharp
// Sort products by name (alphabetical)
public class ProductByNameComparer : IComparer<Product>
{
    public int Compare(Product? x, Product? y)
    {
        if (ReferenceEquals(x, y)) return 0;
        if (x is null) return -1;
        if (y is null) return 1;
        return string.Compare(x.Name, y.Name, StringComparison.OrdinalIgnoreCase);
    }
}

// Sort products by stock (descending)
public class ProductByStockDescendingComparer : IComparer<Product>
{
    public int Compare(Product? x, Product? y)
    {
        if (ReferenceEquals(x, y)) return 0;
        if (x is null) return -1;
        if (y is null) return 1;
        return y.Stock.CompareTo(x.Stock); // reversed for descending
    }
}

// Use different comparers for different scenarios
products.Sort(new ProductByNameComparer());
products.Sort(new ProductByStockDescendingComparer());

// SortedDictionary / SortedList use IComparer in their constructor
var byName = new SortedDictionary<Product, int>(new ProductByNameComparer());
```

## Comparer\<T\> Base Class: The Right Way to Implement IComparer

Instead of implementing the interface directly, derive from `Comparer<T>`:

```csharp
// Comparer<T> provides:
// 1. A default IComparer<T> via Comparer<T>.Default
// 2. Non-generic IComparer implementation (for legacy APIs) for free
// 3. Null safety handling if you implement the null checks

public class ProductByPriceComparer : Comparer<Product>
{
    // Only need to implement this one method
    public override int Compare(Product? x, Product? y)
    {
        if (ReferenceEquals(x, y)) return 0;
        if (x is null) return -1;
        if (y is null) return 1;
        return x.Price.CompareTo(y.Price);
    }
}

// Alternatively, use Comparer<T>.Create for one-liners
IComparer<Product> byPriceDesc = Comparer<Product>.Create(
    (x, y) => y.Price.CompareTo(x.Price));

products.Sort(byPriceDesc);
```

## LINQ OrderBy and Comparison

```csharp
// LINQ OrderBy without key: uses IComparable<T>
var sorted = products.OrderBy(x => x); // requires Product : IComparable<Product>

// LINQ OrderBy with key selector: uses Comparer<TKey>.Default (no IComparable needed on Product)
var byPrice = products.OrderBy(p => p.Price);      // Price is decimal — IComparable<decimal>
var byName = products.OrderBy(p => p.Name);         // Name is string — IComparable<string>
var byStock = products.OrderByDescending(p => p.Stock);

// LINQ OrderBy with IComparer
var byNameComparer = products.OrderBy(p => p, new ProductByNameComparer());

// Multi-level sort
var sorted2 = products
    .OrderBy(p => p.Price)
    .ThenByDescending(p => p.Stock)
    .ThenBy(p => p.Name);
```

<Callout type="senior">
**Senior Insight: StringComparer and Culture-Aware Sorting**

When sorting strings, always specify the comparison type. The default `string.CompareTo()` uses `CurrentCulture` — which changes behavior based on the server's locale settings:

```csharp
// These produce different orderings depending on OS locale:
var words = new[] { "apple", "Banana", "cherry", "Date" };

words.OrderBy(w => w);
// CurrentCulture: might sort by locale rules (Banana before apple in some cultures)

// For consistent, culture-agnostic sorting (identifiers, keys, codes):
var stable = words.OrderBy(w => w, StringComparer.OrdinalIgnoreCase);
// apple, Banana, cherry, Date — consistent on any machine

// For user-facing alphabetical display (UI lists):
var display = words.OrderBy(w => w, StringComparer.CurrentCulture);

// SortedDictionary with explicit comparer — prevents locale bugs
var dict = new SortedDictionary<string, int>(StringComparer.OrdinalIgnoreCase);
dict["Apple"] = 1;
dict["apple"] = 2; // updates the same key — case-insensitive

// The StringComparer.Ordinal family:
// StringComparer.Ordinal            — case-sensitive, byte-order
// StringComparer.OrdinalIgnoreCase  — case-insensitive, byte-order (fast)
// StringComparer.InvariantCulture   — linguistic, culture-independent
// StringComparer.CurrentCulture     — matches user's OS locale (for UI)
```

A subtle production bug: using `List<string>.Sort()` (CurrentCulture) on a server with Turkish locale causes `i` and `I` to sort differently than expected — the infamous "Turkish I problem". Always specify `StringComparer.OrdinalIgnoreCase` for identifier sorting.
</Callout>
